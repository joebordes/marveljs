Deciding what to do.

The first step was to read and learn about the alternatives and decide what we need and which solutions to evaluate.
Apparently a lot can be done using pure CSS but, in the end some javascript is needed and it looks like mostly that is all jQuery direct DOM manipulation based. I am trying to stay away from that.
This site sums up the different approaches:
  • flip/scroll Simply scroll the table in the view and have the rest of the site be responsive (mandatory if comparing between rows is needed). Optionally, you can flip or move the header to the first column and scroll the rest, also making the first column fixed as Zurb Foundation's approach
  • list Convert the table into a one column list (if no comparing between rows is needed)
  • hide Convert the table into a one column list (if no comparing between rows is needed)
Mostly David Bushell's comment sums it up:
One thing that quickly becomes obvious is that there is no single solution. The ideas above may be perfectly adequate for some cases and completely useless for others.
Since "roll-your-own" is not an option I will not be trying the direct CSS approach, and a pure CSS solution I think will be lacking. Since, I like the Filament TableSaw approach over Zurb, I will evaluate Tablesaw and RWD-Table-Patterns to decide which to use.
Our needs are basically non-comparitive, so I will look for a hybrid approach between list and hiding
